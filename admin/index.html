Guia para Adicionar Novos Artigos

O processo para adicionar novos artigos agora é bem mais simples e direto, totalmente integrado ao seu fluxo de trabalho com o GitHub. Você não precisa mais do Netlify CMS. Aqui está o passo a passo detalhado:

Aviso Importante: A pasta para os artigos deve se chamar artigos (sem o sublinhado). Pastas que começam com _ (como _artigos) são frequentemente ignoradas por serviços de hospedagem como o Netlify, o que impede que os artigos sejam publicados. Por favor, certifique-se de usar a pasta /artigos/.

Passo 1: Crie o arquivo do artigo

  1. No seu projeto (no seu computador ou diretamente no GitHub), navegue até a pasta /artigos/.
  2. Dentro desta pasta, crie um novo arquivo de texto. O nome do arquivo será usado na URL. Por exemplo, se você criar meu-novo-artigo.md, o link será .../artigos/meu-novo-artigo. Use nomes simples, em letras minúsculas e separe as palavras com hífens.
  3. A extensão do arquivo deve ser .md (Markdown).

Passo 2: Adicione o conteúdo e os metadados

Abra o novo arquivo .md que você criou e cole o modelo abaixo. Este bloco inicial, entre os ---, é chamado de "frontmatter" e contém todas as informações do seu artigo.

---
title: "Aqui vai o título completo do seu artigo"
publish_date: "2024-08-15T10:00:00.000Z"
author: "Seu Nome"
image: "https://url-da-imagem.com/imagem.jpg"
summary: "Um resumo curto e interessante que aparecerá no card da lista de artigos. Idealmente, 1 a 2 frases."
---

Aqui você começa a escrever o conteúdo do seu artigo.

## Use subtítulos para organizar
Você pode usar a sintaxe Markdown para formatar seu texto. Por exemplo, dois sinais de hash (##) criam um subtítulo.

- Use um hífen para criar listas com marcadores.
- **Dois asteriscos** deixam o texto em **negrito**.

> E um sinal de maior cria um bloco de citação, para destacar informações importantes.

É simples assim!

Preencha os campos:

  • title: O título que aparecerá no topo da página do artigo.
  • publish_date: A data de publicação. Mantenha o formato AAAA-MM-DD seguido de T10:00:00.000Z. Apenas altere o ano, mês e dia.
  • author: O nome do autor do post.
  • Muito importante! image: Cole aqui o link direto (URL) para a imagem de destaque do seu artigo. Você pode usar um serviço como o Unsplash para encontrar imagens gratuitas e de alta qualidade.
  • summary: O pequeno texto que vai aparecer no card do artigo.

Abaixo da segunda linha de ---, escreva o texto completo do seu artigo.

Passo 3: Atualize o "índice" de artigos

Este é o passo final e mais importante para que seu artigo apareça na lista.

  1. Abra o arquivo /artigos/index.json.
  2. Você verá uma estrutura parecida com esta: 
    [
  {
    "slug": "artigo-de-exemplo",
    "title": "Entendendo a Perícia Médica: Um Guia para Pacientes",
    "publish_date": "2024-07-28T10:00:00.000Z",
    "author": "Dr. José Luiz de Souza Neto",
    "image": "https://images.unsplash.com/photo-1576091160550-2173dba999ef?...",
    "summary": "Navegue pelo processo da perícia médica..."
  }
]
  3. Adicione um novo bloco de informações para o seu novo artigo, copiando os dados que você colocou no "frontmatter". Não se esqueça de adicionar uma vírgula após o bloco do artigo anterior se houver mais de um.

Exemplo: Adicionando meu-novo-artigo.md:

[
  {
    "slug": "meu-novo-artigo",
    "title": "Aqui vai o título completo do seu artigo",
    "publish_date": "2024-08-15T10:00:00.000Z",
    "author": "Seu Nome",
    "image": "https://url-da-imagem.com/imagem.jpg",
    "summary": "Um resumo curto e interessante..."
  },
  {
    "slug": "artigo-de-exemplo",
    "title": "Entendendo a Perícia Médica: Um Guia para Pacientes",
    "publish_date": "2024-07-28T10:00:00.000Z",
    "author": "Dr. José Luiz de Souza Neto",
    "image": "https://images.unsplash.com/photo-1576091160550-2173dba999ef?...",
    "summary": "Navegue pelo processo da perícia médica..."
  }
]

Passo 4: Envie para o GitHub

Salve os dois arquivos (seu-novo-artigo.md e index.json) e envie as alterações (faça o "commit" e "push") para o seu repositório no GitHub. O deploy automático que você já configurou fará o resto, e em poucos minutos seu novo artigo estará no ar!

Pronto! É um processo manual, mas que lhe dá total controle e é muito mais robusto e simples de manter a longo prazo.
